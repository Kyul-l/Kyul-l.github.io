A04와 A05는 각각 '데이터가 평문으로 노출되는 이유'와 '신뢰할 수 없는 입력이 명령어로 실행되는 이유'를 다룬다. 둘 다 2017년부터 존재했지만, 명칭과 범위의 변화가 문제를 바라보는 시각을 어떻게 바꿨는지가 핵심이다.

## A04 — 암호화 실패 (Cryptographic Failures)

2021년에 '민감 데이터 노출'에서 '암호화 실패'로 명칭이 바뀐 것은 단순한 리브랜딩이 아니다. 전자는 결과(데이터가 노출됨)를 설명하고, 후자는 원인(암호화 메커니즘이 잘못되었거나 부재함)을 지목한다. 이 전환이 중요한 이유는, 증상을 수정하는 것과 원인을 수정하는 것이 완전히 다른 접근을 요구하기 때문이다.

**하드코딩된 암호화 키와 형상 관리 유출(CWE-547)** 은 가장 흔하면서도 파급력이 큰 사례다. 개발자가 편의를 위해 소스 코드·구성 파일·CI/CD 스크립트 내부에 API 키, 데이터베이스 암호화 키, 서비스 계정 비밀번호를 직접 작성한다. 코드가 내부 Git 저장소에서 유출되거나 클라이언트 측 JavaScript에 포함되어 배포되면, 공격자는 역공학(Reverse Engineering)을 통해 키를 추출하고 시스템에 저장된 모든 암호화 데이터를 복호화할 수 있다. 암호화가 존재해도 키 관리가 잘못되면 없는 것과 같다.

**취약한 전송 계층 보호(CWE-315)** 는 HTTPS를 강제하지 않거나, MD5·SHA-1 같은 이미 폐기된 해시 알고리즘과 취약한 암호 스위트를 유지하는 경우다. 동일한 네트워크 내에서 패킷을 스니핑할 수 있는 공격자는 세션 쿠키나 평문 자격 증명을 가로챈다. 클라이언트가 HTTPS로 요청을 시작해도 서버가 HTTP 혼합 콘텐츠를 허용하거나 HSTS(HTTP Strict Transport Security)를 적용하지 않으면 다운그레이드 공격에 취약하다.

암호화 실패의 방어는 '암호화 적용'이 아니라 '올바른 암호화의 전체 수명 주기 관리'에 있다. 알고리즘 선택, 키 생성·저장·로테이션, 전송 레이어 강제, 폐기 알고리즘의 명시적 금지까지 포괄한다.

## A05 — 인젝션 (Injection)

인젝션은 2017년 1위에서 2025년 5위로 하락했다. 그러나 발생 빈도가 줄었다는 의미는 아니다. XSS가 2021년에 인젝션 범주로 흡수되면서 순위가 분산되었고, 파라미터화된 쿼리(Parameterized Query)와 ORM의 보급이 일부 패턴을 줄였을 뿐, SQL 인젝션 관련 CVE는 여전히 매년 최다 발생 취약점 중 하나다.

인젝션의 본질은 단순하다. 애플리케이션이 신뢰할 수 없는 외부 입력을 정제하지 않고 데이터베이스 쿼리나 OS 명령어, 인터프리터에 그대로 전달하는 것이다. 데이터와 명령어를 구분하지 못한 결과다.

**SQL 인젝션(SQLi)** 의 고전적 패턴은 로그인 창에서 사용자 ID 필드에 `admin' --` 또는 `' OR '1'='1`을 입력하는 것이다. 애플리케이션이 이 입력을 문자열 결합으로 쿼리에 직접 삽입하면, `WHERE username='admin'--'` 처럼 뒷부분이 주석 처리되어 비밀번호 확인 로직이 무력화되고 관리자 권한이 부여된다. 파라미터화된 쿼리나 ORM을 사용하면 입력값이 항상 데이터로만 처리되어 이 패턴이 차단된다.

**OS 명령어 인젝션** 은 더 치명적이다. 네트워크 진단 도구처럼 사용자 입력(IP 주소)을 받아 백엔드에서 `ping` 명령어를 실행하는 기능이 있을 때, 공격자가 `127.0.0.1; cat /etc/passwd`를 입력하면 세미콜론 이후의 추가 명령어가 함께 실행된다. 서버가 이를 필터링 없이 셸에 전달하면 운영체제 수준의 제어권이 공격자에게 넘어가고, 이후 악성 스크립트를 다운로드하여 실행하는 것은 단순한 연쇄 작업이 된다.

XSS(크로스 사이트 스크립팅)은 대상이 서버가 아닌 브라우저라는 점만 다르다. 악성 스크립트가 신뢰할 수 있는 도메인의 페이지에 삽입되어 다른 사용자의 브라우저에서 실행될 때, 해당 사용자의 세션 토큰을 탈취하거나 DOM을 조작하는 것이 가능해진다.

방어의 핵심은 입력을 '신뢰의 경계에서' 검증하고 인코딩하는 것이다. 출처에 관계없이 모든 외부 입력을 잠재적 위협으로 다루는 것이 출발점이다.
