OWASP(Open Web Application Security Project)는 비영리 재단으로, 실제 침해 사례와 CVE 데이터를 기반으로 웹 애플리케이션 10대 보안 위협을 주기적으로 발표한다. 2025년 버전은 175,000건 이상의 CVE와 589개의 CWE 분류 데이터를 바탕으로 작성되었다. 단순한 코드 결함 목록에서 출발해, 현재는 클라우드 인프라·소프트웨어 공급망·시스템 복원력까지 범위를 포괄하는 프레임워크로 진화했다.

세 번의 릴리스를 거치며 목록의 철학 자체가 변했다. 2017년은 XSS, XXE, 역직렬화 같은 개별 공격 기법이 독립 항목으로 나열된 '증상 중심' 목록이었다. 2021년은 이를 상위 개념으로 통합해 '근본 원인' 중심으로 재편했다. 2025년은 한 걸음 더 나아가 공급망 생태계 전체와 예외 상황에서의 시스템 복원력을 보안의 책임 범위로 끌어들였다.

## 10대 항목 개요

**A01 — 취약한 접근 제어** 인증된 사용자가 허용되지 않은 리소스에 접근할 수 있을 때 발생한다. BOLA(객체 수준 인가 결함)·BFLA(기능 수준 인가 결함)·SSRF(서버 사이드 요청 위조)가 대표 패턴이다. REST API와 마이크로서비스 환경에서 가장 빈번하게 악용된다.

**A02 — 보안 설정 오류** 코드가 완벽해도 인프라 설정 하나로 시스템 전체가 노출될 수 있다. 공개된 클라우드 스토리지, 운영 환경에 남겨진 디버그 모드, 기본 자격증명이 전형적 사례다. 테스트된 애플리케이션의 100%에서 어떤 형태로든 발견될 만큼 광범위하다.

**A03 — 소프트웨어 공급망 장애** 서드파티 라이브러리·빌드 인프라·배포 파이프라인 전체가 공격 대상이다. Dependency Confusion처럼 타겟 애플리케이션이 아닌 그것이 신뢰하는 상위 공급자를 노리는 패턴이 표준화된 위협이 됐다.

**A04 — 암호화 실패** 민감 데이터가 암호화되지 않거나, 폐기된 알고리즘(MD5·SHA-1)을 사용하거나, 키를 코드에 하드코딩할 때 발생한다. '민감 데이터 노출'에서 '암호화 실패'로 명칭이 바뀐 것은 증상이 아닌 원인에 집중하겠다는 전환이다.

**A05 — 인젝션** SQL·XSS·OS 명령어·NoSQL 인젝션을 모두 포괄한다. 2017년부터 XSS가 흡수되며 순위는 하락했지만 CVE 발생 빈도는 여전히 최상위다. 신뢰할 수 없는 외부 입력을 검증 없이 인터프리터에 전달하는 구조적 오류다.

**A06 — 안전하지 않은 설계** 구현이 아닌 설계 단계의 결함이다. 위협 모델링의 부재와 비즈니스 로직의 취약한 흐름이 원인이다. 가격 조작·결제 플로우 우회 같은 공격은 코드를 완벽하게 작성해도 막을 수 없다.

**A07 — 인증 실패** MFA 부재, 취약한 비밀번호 복구 흐름, 세션 관리 결함이 포함된다. 크리덴셜 스터핑은 유출된 수백만 쌍의 조합을 자동화로 시도하며, Rate Limiting이 없으면 대규모 계정 탈취로 이어진다.

**A08 — 소프트웨어·데이터 무결성 오류** 데이터나 코드가 변조되지 않았음을 검증하는 장치가 없을 때 발생한다. 디지털 서명 확인 없이 처리되는 역직렬화 객체, 무비판적으로 수용한 외부 플러그인이 대표 사례다.

**A09 — 보안 로깅·알림 실패** 로그가 존재해도 실시간 알림과 연동되지 않으면 방치된 것이나 다름없다. 2025년에 '모니터링'에서 '알림'으로 명칭이 강화된 것은 가시성(Visibility)만으로는 부족하다는 판단이다.

**A10 — 예외 처리 오류** 2025년 신규 항목. 예외 상황에서 자원을 제어하지 못하거나, 검증 실패를 '안전'으로 처리하는 Fail-Open 결함, 또는 불완전한 트랜잭션 롤백이 초래하는 데이터 무결성 파괴를 다룬다.
